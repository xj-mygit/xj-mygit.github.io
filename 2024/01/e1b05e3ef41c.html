# 镜像命令

# Docker images

列出 == 本地主机 == 上的镜像。

OPTIONS：

  • -a：列出所有的镜像 ==（含历史镜像层）
  • -q：只显示镜像 id

字段说明：
REPOSITORY：镜像的仓库源
TAG：镜像的标签版本号
IMAGE ID： 镜像的 ID
CREATED： 镜像的创建时间
SIZE：镜像的大小
同一个仓库源可以有多个 TAG 版本。使用 REPOSITORY:TAG 可以定义指定版本的仓库源。如不指定则默认最新。

# Option 说明

模板： docker search option 镜像
– limit：设置搜索的结果数量，默认是 25 个。

意思就是：查询 mysql 镜像当前页只展示 10 条记录。

–no-trunc: 不截断输出。因为 Description 有的会很长。所以 docker 默认缩略显示以适用于相对应的宽度。

– format ：以模板的形式输出内容。
注意点：暂时只发现可以输出 name 和 desctiption

-f ,–filter: 目前只能测试出过滤星数

# Docker pull

下载镜像
Docker Pull 某个镜像名称 :TAG

注意：

  • 不携带版本后缀的，默认下载 latest 标签的版本。
  • latest 标签的不一定是最新版本

# Docker system df

查询镜像 / 容器 / 数据卷的数量，空间等信息。df（disk file）

TYPE：

  • Images： 镜像
  • Containers： 容器
  • Local Volumes： 数据卷
  • Build Cache： 缓存
    TOTAL： 数量
    ACTIVE： 活动中的
    SIZE：大小
    RECLAIMABLE： 可伸缩性。

# Docke rmi 镜像名称 /id

删除镜像。

删除多个：docker rmi 镜像 1 镜像 2
删除全部：docker rmi $

注意：如果遇到镜像生成了容器，可以使用 -f 强制删除。force。
如 docker rmi -f 镜像 1

# 新建 + 启动命令

docker run option image command args
OPTION (常用)

  • –name = “名称”: 指定容器名称。不指定则 docker 自行分配。
  • -d: 后台运行容器并返回容器 ID。即启动守护式容器（后台运行）
  • -i: 以交互模式运行容器，通常和 - t 一起使用。
  • -t: 为容器重新分配一个伪输入终端，通常和 - i 一起使用
  • -it: 启动交互式容器（前端有伪终端，等待交互）
  • 交互式即提供一个伪终端，供于对容器内的 linux 系统做交互。
  • -P: 随机端口映射，
  • -p: 指定端口映射

注意点：

  1. 以交互模式启动容器且进入命令行界面。可能会失败。错误提示如下：
    docker: Error response from daemon: failed to create task for container: failed to create shim task: OCI runtime create failed: runc create failed: unable to start container process: exec: “/bin/sh”: stat /bin/sh: no such file or directory: unknown.
    原因是你启动的镜像的基础 liunx 镜像不携带 /bin/bash 的命令。
  2. 简易理解交互模式：
  3. docker 容器相当于一个 linux 系统。我们对系统中某个应用做一些处理需要使用命令行操作。继而 /bin/bash 可以提供这个平台
  4. RUN 命令是将镜像生成容器。

# 列出容器

Docker ps option

  • -a : 列出正在运行的容器，以及历史运行过的容器 ==> 包括被卸载过的，只要进行过 run 命令的就会出现在 - a 所列出的列表中
  • -q：列出正在运行的容器。只展示容器 id。
  • -i: 显示最近创建的容器
  • -n：显示最近 n 个创建的容器

# 退出容器

Exit/ctrl+p+q

Exit：直接退出容器，且容器停止。
ctrl+p+q：退出容器，容器不停止。

# 重启，启动，停止，强制停止容器

重启：docker restart 镜像 id / 名称
启动： docker start 镜像 id / 名称
停止：docker stop 镜像 id / 名称
强制停止：docker kill 镜像 id / 名称

这里是对已经生成容器的容器做重启，启动等操作

# 删除容器

docker rm 镜像 id / 名称
强制删除：docker rm -f 镜像 id / 名称
一次删除多个：docker rm -f ${docker ps -a -q} 或者 docker ps -a -q | xargs docker rm
