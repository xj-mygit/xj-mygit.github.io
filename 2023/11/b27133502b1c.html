# Docker 简介

# Docker 生成背景

  1. 由于各种配置、开发环境的不同（例如各种版本不一致的问题，mysql、java、redis 等），导致了不同人员使用系统时的各种问题。
  2. 换一台机器就要重新部署一遍，费时费力
    由此衍生出 Docker —— 系统平滑移植，容器虚拟化计划
    Docker 解决了：通过将环境复制到 Docker，解决了我的代码只能在我自己电脑上正常工作的问题。

# Docker 理念

一次镜像，处处运行
Docker 是基于 Go 语言实现的开源项目
Docker 主要目标是：Build、Ship And Run Any App，Anywhere；通过对应用组件的封装、分发、部署、运行等生命周期的管理，使用户的 App（可以是一个 Web 应用或者数据库应用）及其运行环境做到一次镜像，处处运行。

Linux 容器技术出现解决了这样一个问题，Docker 是基于此发展出来的。

解决了：运行环境和配置问题的软件容器，方便做持续集成有助于整体发布的容器虚拟话技术
